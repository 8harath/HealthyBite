# HealthyBite

AI-powered personalized meal recommendation app. Enter your health profile and get 5 region-aware meal suggestions generated by Groq's Llama 3.3 70B — complete with gram-level ingredient quantities, step-by-step cooking instructions, fun facts, and digestion tips for each meal.

## Features

- **Health questionnaire** — 4-step form collecting age, weight, goals, dietary preference, allergies, and location
- **AI meal recommendations** — Groq (Llama 3.3 70B) generates 5 personalized meals with full nutritional info
- **Regional cuisine support** — 21 Indian regions mapped to authentic local dishes and ingredients
- **Detailed meal view** — ingredients with exact gram quantities, cooking steps, extended nutrition (fiber/sugar/sodium), fun facts, and digestion/consumption tips
- **Rule-based fallback** — 100+ curated meals (40 general + regional meals for 12 states) if Groq is unavailable
- **Weekly meal planner** — distribute recommendations across a 7-day calendar with meal swapping
- **Recipe & shopping list generation** — Gemini-powered prototype (separate from main flow)
- **Subscription plans** — weekly / monthly / quarterly tiers with simulated payment
- **Real authentication** — Supabase Auth with session-based cookies and route protection
- **Dark mode** — system-aware with manual toggle
- **Type-safe development** — Full TypeScript coverage with strict checks

## Quick Start

### Prerequisites

- Node.js 22.x, npm 10+
- [Supabase](https://supabase.com) project (free tier works)
- [Groq API key](https://console.groq.com) (free, optional — fallback engine is used if not set)

### Setup

1. Clone and install dependencies:
   ```bash
   git clone <repo-url>
   cd healthybite
   npm install
   ```

2. Copy the environment template and fill in your values:
   ```bash
   cp .env.example .env.local
   ```

3. Run the database schema in your Supabase SQL Editor:
   ```bash
   # Open Supabase Dashboard → SQL Editor → paste contents of supabase/schema.sql
   ```

4. Start the development server:
   ```bash
   npm run dev
   ```

### Environment Variables

| Variable | Required | Description |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Your Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | Your Supabase anon/public key |
| `GROQ_API_KEY` | No | Groq API key for AI recommendations |

> If `GROQ_API_KEY` is not set, the app automatically uses the rule-based fallback engine. All other features work normally.

## How It Works

```
Register / Login
      ↓
4-Step Questionnaire
(age, goals, diet, allergies, location)
      ↓
POST /api/profile → Supabase health_profiles
      ↓
POST /api/recommendations
      ├─ Groq (Llama 3.3 70B) → 5 regional meals with
      │   ingredients, instructions, fun facts, tips
      └─ Fallback engine → rule-based selection from 100+ meals
      ↓
Stored in Supabase recommendations table (JSONB)
      ↓
Recommendations page → Meal Detail page
```

## Regional Cuisine Coverage

The AI prompt maps 21 Indian regions to their authentic cuisine context:

- **South India**: Tamil Nadu, Kerala, Karnataka, Andhra Pradesh, Telangana, Goa
- **North India**: Delhi/NCR, Punjab, Uttar Pradesh, Rajasthan, Himachal Pradesh, Jammu & Kashmir
- **East India**: West Bengal, Odisha, Bihar, Assam, Northeast, Uttarakhand
- **West/Central**: Maharashtra, Gujarat, Madhya Pradesh, Chhattisgarh

The fallback engine additionally has curated regional meals for 12 states.

## Docker

```bash
  --build-arg NEXT_PUBLIC_SUPABASE_URL=<your-url> \
  --build-arg NEXT_PUBLIC_SUPABASE_ANON_KEY=<your-key> \
  -t healthybite .
docker run -p 3000:3000 --env-file .env.local healthybite
```

Or with Docker Compose:

```bash
docker-compose up
```

## Current Limitations

- Rate limiting is in-memory and resets on restart (not suitable for horizontal scaling)
- Subscription and payment flows are simulated — no real payment processing
- Gemini integration (meal planning, recipes, shopping lists) is prototype-only with no auth or DB persistence
- The weekly meal planner generates plans from cached recommendations only
- No email sending for forgot-password in local dev (requires Supabase SMTP config)
